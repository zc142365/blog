How I Created a Blog

2023-09-01
Uncategorized
688 words

Why the Framework?

To create a blog quickly, I looked for a framework that made it easy to create a static blog.

Found and selected frameworks

Framework type

Jekyll, hugo, hexo

framework of your choice

There are many themes and I chose hexo because it contained the theme I wanted.

hexo installation process

  1. Install node.js and npm (required)
  • Installer download address : https://nodejs.org/
  • I used node 18.17.1 version
  1. Install another package manager (optional)
    I used cmd in a windows environment
  • Entered command
    
  npm install -g pnpm
    Restart terminal
    
  pnpm setup
  1. Installing and using hexo

  • Since I installed pnpm, I will explain based on pnpm.

  • hexo installation address : https://hexo.io/docs/index.html

    
  pnpm install -g hexo-cli

  • Restart terminal and create hexo project

    
  hexo init blog

  • Install required packages for hexo project with pnpm from project

    
  cd blog
  pnpm install

  • Find a theme to apply to your hexo project
    address : https://hexo.io/themes/index.html


    I chose vivia among several themes. 
    
  git clone https://github.com/saicaca/hexo-theme-vivia themes/vivia
  cd themes
  pnpm install
  cd ../

  • Theme settings need to be written in _config.yml inside the project.

    
  theme: vivia

  • The vivia theme requires other settings, so _config.vivia.yml must also be written.

    
  # Header
  menu:
  Home: /
  Archives: /archives
  About: /about
  rss: /atom.xml

  home:
  style: default           # default, detail

  favicon:                   # Path to the custom favicon (e.g. "/favicon.png" if your favicon file is located at /source/favicon.png)

  # Banner
  banner:
  enable: false            # Display banner
  url: /assets/banner.png
  position: center         # Specifies the alignment of the image, see the "object-position" property in CSS
  onAllPages: true         # Display banner on all pages

  # Appearence
  hue: 270                   # The hue of the theme color (e.g. red: 0, orange: 60, blue: 260, purple: 300, pink: 345)

  # Sidebar
  sidebar:
  widgets:                 # Plugins to be enabled (profile, category, tag, archive, recent_posts)
      normal:                # Scroll along with the page
      - profile
      sticky:                # Stick to the top of the page
      - category
      - tag
      - archive
      - recent_posts

  # Widget behavior
  archive_type: 'monthly'
  recent_posts_limits: 5

  # Personal info
  avatar: 
  author: Your Name
  subtitle: This is the subtitle
  links:
  - name: Twitter
      icon: fa-brands fa-twitter      # Find icon codes at https://fontawesome.com/search
      url: https://twitter.com
  - name: Steam
      icon: fa-brands fa-steam
      url: https://store.steampowered.com
  - name: GitHub
      icon: fa-brands fa-github
      url: https://github.com

  comment:
  valine:
      enable: false
      appId: 
      appKey: 
      placeholder: Just go go
      pageSize: 10
      highlight: true
      serverURLs: 
      # Other supported parameters can be added here if needed
      # Visit https://valine.js.org/configuration.html for more details


  #Analytics
  gauges_analytics: false
  google_analytics: false #UA-xxxxxx
  baidu_analytics: false # ddxxxxxxxxxxxxxxxxx In the https://hm.baidu.com/hm.js? Afterwards

  #Copyright
  copyright: #

  • After creating the page, deploy the hexo project.
    And the hexo command can be found at https://hexo.io/docs

    
  hexo new page about
  hexo clean
  hexo gen
  hexo server

  • After entering the above command, deployment is completed at localhost:4000 and connection is possible.
    The above must be done after creating a repository on github.
    If you have confirmed that the server has been deployed well, set it in _config.yml to deploy to github.

    
  url: https://Myname.github.io

  deploy:
      type: 'git'
      repo: https://github.com/Myname/myname.github.io
      branch: main

  • This command is a hexo command for deployment and should be performed assuming that you have completed setting _config.yml and set up a github account.

    
  hexo deploy

  • If you’ve made it this far, distribution will have been successful.
